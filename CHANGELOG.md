# min-agent-demo 版本记录

这个文件记录每个版本对使用者带来的变化，而不是代码改动细节。

## V3 - 多文件上下文读取

状态：已完成

这一版解决什么问题：

V2 已经可以让真实模型判断下一步，但上下文仍主要来自单个文件。V3 让 Agent 可以先查看 workspace 目录，再选择相关 Markdown 文件读取并综合回答。

使用者能感受到什么：

- 可以让 Agent 总结整个示例工作区，而不必指定单个文件。
- 观察窗口能看到 Agent 先查看目录，再读取多个文件。
- FakeLLM 模式仍可离线演示多文件上下文流程。
- DeepSeek 模式可以在 `list_dir` 和 `read_file` 之间选择下一步。

这一版不会做什么：

- 不会让 Agent 修改文件。
- 不会让 Agent 执行命令。
- 不会引入长期记忆。
- 不会让网页控制 Agent。

怎么判断这一版完成：

- `list_dir` 能安全列出 workspace 内目录。
- FakeLLM 可以在未指定文件名时完成目录发现、多文件读取和综合回答。
- DeepSeek prompt 允许模型选择 `list_dir` 和 `read_file`。
- 所有测试通过。

## V2 - 接入真实模型判断能力

状态：已完成

这一版解决什么问题：

V1 只能用预设规则演示 Agent 判断。V2 接入 DeepSeek，让 Agent 可以由真实模型判断下一步，同时保留本地工具边界。

使用者能感受到什么：

- 可以选择使用真实模型参与 Agent 决策。
- 默认 FakeLLM 模式仍可离线运行。
- 观察窗口仍能看到 Agent 每一步。
- 模型调用失败时，系统会给出可理解的失败说明，而不是崩溃。

这一版不会做什么：

- 不会让 Agent 修改文件。
- 不会让 Agent 执行命令。
- 不会增加复杂工作台。
- 不会让网页控制 Agent。

怎么判断这一版完成：

- 使用 FakeLLM 仍能正常运行。
- 配置 DeepSeek key 后，Agent 可以用真实模型完成读取并总结文件。
- 模型失败时，系统不会崩溃，而是给出可理解的失败说明。

## V1 - 最小可观察 Agent 闭环

状态：已完成

这一版解决什么问题：

先证明一个最小 Agent 可以跑起来，并且用户能看到它每一步为什么发生。

使用者能感受到什么：

- 输入一个简单任务后，系统会读取示例文件并给出总结。
- 浏览器观察窗口会展示 Agent 的执行过程。
- 每次运行都会保存记录，便于复盘。
- 文件读取被限制在指定 workspace 内，避免误读其他文件。

这一版不会做什么：

- 不接真实大模型。
- 不修改文件。
- 不执行命令。
- 不支持复杂任务。

怎么判断这一版完成：

- 可以启动 CLI。
- 可以打开观察窗口。
- 可以看到任务从开始到完成的完整过程。
- 缺失文件或越界路径不会导致程序崩溃。

## 后续可能方向

### V4 候选 - 更友好的观察和讲解体验

让观察窗口更清楚地展示每一步属于哪个 Agent 模块、为什么发生、输入输出是什么，方便演示和复盘。

这一版仍不控制 Agent，不新增危险动作。

### V5 候选 - 引入可控的写文件权限

在明确确认机制下，让 Agent 可以提出写文件动作，并在用户批准后执行。

这一版先不运行命令。

### V6 候选 - 运行记忆和项目状态

让 Agent 能复用历史运行记录、保存任务摘要，并在观察窗口中说明本次使用了哪些历史信息。

这一版不做复杂长期记忆系统，不自动改写项目规则。

### V7 候选 - 引入可控的命令执行

在更严格的权限确认下，让 Agent 可以提出运行命令的动作，并展示命令、原因、风险和执行结果。

这一版只考虑白名单命令、固定 workspace、超时控制和输出截断，不开放任意 shell。

### V8 候选 - 本地工具扩展机制

让 demo 支持用简单声明注册本地工具，观察 Agent 如何发现工具、选择工具和调用工具。

这一版不接复杂 MCP 生态，不做插件市场。

### V9 候选 - 简单任务分工

让 Agent 能把一个目标拆成少量可观察的子任务，并按顺序执行和汇总结果。

这一版只演示任务分解机制，不做真正多 Agent 编排。
